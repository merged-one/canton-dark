# Plan Template

Use this template for feature plans, migration notes, and multi-step implementation prompts.

## Objective

- What outcome must be achieved?

## Constraints

- Regulatory, product, architectural, or delivery constraints that must remain true.

## Existing Context

- Relevant files, packages, ADRs, and current behavior.

## Proposed Plan

1. Discovery and reconciliation steps.
2. Implementation steps by package or workflow.
3. Verification steps, including deterministic test coverage.

## Risks And Open Questions

- Explicit technical or product risks.
- Open questions that block safe implementation.

## Verification

- Exact commands to run.
- Expected evidence of success.

## Follow-Up

- Intentionally deferred work.
